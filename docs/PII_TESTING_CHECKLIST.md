# Чеклист тестирования PII-фильтра

**Дата тестирования:** 24.04.2026
**Основание:** ДС №1 к Договору 2703/26-01, п. 4.2.2
**Состав тестеров:** до 4 человек от Заказчика + команда Исполнителя

## Модель защиты

**Защищаем:** данные, которые прислал ПОЛЬЗОВАТЕЛЬ (клиент магазина) в своём сообщении. Они не должны попасть:
- в запрос к LLM (Anthropic/DeepSeek/…)
- в запрос к внешнему поиску (Tavily)
- в JSONL-логи бота

**Не трогаем:** публичную информацию — телефоны горячих линий Евроопта, адреса магазинов, ФИО генерального директора. Бот свободно их использует в ответах и в контексте (RAG/web).

## Что маскируется (только в сообщениях пользователя)

| Тип | Плейсхолдер | Примеры |
|---|---|---|
| Телефон | `[телефон]` | `+375 29 123-45-67`, `8029 555 44 33`, `+7 495 123 45 67` |
| Email | `[email]` | `ivan@mail.ru`, `info@belhard.com` |
| Банковская карта | `[номер_карты]` | `4111 1111 1111 1111` (проверка Luhn) |
| Карта лояльности | `[номер_карты]` | `карта лояльности 1234567890123` |
| Паспорт РБ | `[паспорт]` | `MP1234567`, `HB 9876543` |
| ID РБ | `[ID]` | `1234567A123PB1` |
| Физ. адрес | `[адрес]` | `ул. Ленина, д. 5, кв. 12` |
| ФИО | `[ФИО]` | `Иван Петров`, `Мамоненко Игорь Викторович` |
| Дата рождения | `[дата]` | `дата рождения 15.03.1985` |

## Сценарии для отправки в бот

### A. Положительные сценарии (ПДн должны исчезнуть из ответа и логов)

1. `Меня зовут Иван Петров, позвоните на +375 29 123-45-67`
2. `Мой email ivan.test@mail.ru, когда доставка?`
3. `У меня карта 4111 1111 1111 1111, как оплатить?`
4. `Паспорт MP1234567, скидка будет?`
5. `Доставьте на ул. Ленина, д. 5, кв. 12`
6. `Карта лояльности 1234567890123, сколько баллов?`
7. `Я Влад Сидоров, акции сегодня?` — проверка NER (имя без суффикса)
8. `Дата рождения 15.03.1985, будут ли скидки именинникам?`

**Ожидается в ответе бота:** ваши ПДн не повторяются в ответе (бот физически не видел их — он получил плейсхолдеры). Бот отвечает по сути вопроса либо просит переформулировать без личных данных. Если бот сам назовёт телефон горячей линии, адрес магазина или имя публичного лица — это нормально (это не ваши ПДн).

### B. Отрицательные сценарии (ложных срабатываний быть не должно)

9. `Какие акции в Евроопте сегодня?`
10. `Где ближайший Грошык?`
11. `Как получить карту Еплюс?`
12. `Рецепт драников, пожалуйста`
13. `Работает ли магазин в воскресенье?`
14. `Скидки на мясо сейчас есть?`

**Ожидается:** бот отвечает нормально, ничего не маскирует (плейсхолдеры не появляются).

### C. Комбинированные кейсы

15. `Я Иван Петров, тел +375 29 123-45-67, email ivan@mail.ru, доставить на ул. Ленина, д. 5` — в ответе не должно быть ни одного из этих данных.
16. `Мой номер 80291234567 карта 4532015112830366 паспорт MP1234567` — все три типа должны исчезнуть.

## Как проверить результаты

### 1. Визуально — в чате бота

- После отправки сообщения посмотреть в ответе бота: **ни одна цифра/буква из ваших ПДн не должна повториться** (конкретно ваш телефон, ваш email, ваше имя).
- **Публичные данные** в ответе — это ожидаемо и правильно: телефон горячей линии «+375 17 239-00-00», адрес магазина «ул. Казинца, 52», имя «Кукурузин Владимир Юрьевич» и т.п.

### 2. В JSONL-логах (для команды Исполнителя)

Логи пишутся в `/app/persist/logs/interactions_YYYY-MM-DD.jsonl` (prod) или `./logs/` (локально).

**Проверка через grep:**

```bash
# Сырой телефон +375 29 123-45-67 НЕ должен встречаться в логах
grep -F "+375 29 123" logs/interactions_2026-04-24.jsonl && echo "FAIL: сырой телефон в логе" || echo "OK"

# Email НЕ должен встречаться
grep -F "ivan.test@mail.ru" logs/interactions_2026-04-24.jsonl && echo "FAIL: email в логе" || echo "OK"

# Плейсхолдер [телефон] ДОЛЖЕН встречаться (факт маскирования)
grep -F "[телефон]" logs/interactions_2026-04-24.jsonl && echo "OK" || echo "FAIL: нет плейсхолдера"

# Типы сработавших фильтров
grep -o '"pii_detected_input": \[[^]]*\]' logs/interactions_2026-04-24.jsonl | sort | uniq -c
```

### 3. Автопроверка — Python-скрипт

```bash
cd /path/to/dev
python3.11 scripts/pii_smoke_check.py
```

Вывод: таблица из ~35 кейсов и строка `Итог: 35/35 прошли, 0 упали.`

### 4. Unit-тесты (CI)

```bash
python3.11 -m pytest tests/test_pii_filter.py tests/test_pipeline_integration.py::TestPipelinePIIMasking -v
```

Должно быть: 67 passed, 0 failed.

## Известные ограничения v1

- **Regex-адреса** требуют маркер (`ул.`, `пр.`, `д.`). «Казинца 52» без «ул.» не будет замаскировано.
- **NER** может пропустить редкие иностранные имена (Natasha обучена на русскоязычных новостях).
- **Карты лояльности без маркера** — чисто цифровая последовательность 10–13 цифр без слова «карта» может не сработать.
- **Имена-омонимы** (Мария, Анастасия) — добавлены в стоп-слова, чтобы не маскировать нарицательные употребления; если тестер напишет «меня зовут Мария» — ФИО отловится по маркеру «меня зовут», если просто «Мария купила хлеб» — нет.

## Что делать при FAIL

1. Записать точный текст сообщения и полученный ответ бота.
2. Приложить строку из JSONL-лога (если сырые данные попали туда).
3. Открыть тикет в TaskMaster / описать в чате команды.
