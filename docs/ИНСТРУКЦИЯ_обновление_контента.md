# Инструкция по обновлению контента AI-помощника

**Проект:** AI-помощник для покупателей Евроопт
**Версия:** 1.0
**Дата:** 07 апреля 2026

---

## Общая информация

AI-помощник работает с тремя типами контента:

| Тип контента | Как часто обновлять | Кто отвечает |
|-------------|---------------------|--------------|
| **Акции** | Еженедельно (по мере обновления) | Маркетинг |
| **FAQ** | По мере появления новых вопросов | Поддержка / маркетинг |
| **Рецепты** | По необходимости | Маркетинг |

Все данные хранятся в формате JSON в папке `data/`. После обновления файлов бот подхватывает изменения при перезапуске.

## 1. Обновление акций

### Где хранятся
```
data/promotions/
├── sample.json              — тестовые акции
└── edostavka_crush_price.json — акции «Крашцены» с edostavka.by
```

### Формат файла
Создайте файл `data/promotions/акции_ДДММГГ.json`:

```json
[
  {
    "name": "Куриное филе охлаждённое",
    "old_price": "12.90",
    "new_price": "8.90",
    "end_date": "2026-04-15",
    "category": "мясо",
    "description": "Акция во всех магазинах сети"
  },
  {
    "name": "Молоко Простоквашино 3.2%",
    "old_price": "3.50",
    "new_price": "2.80",
    "end_date": "2026-04-15",
    "category": "молочные",
    "description": "1 литр, только с картой Еплюс"
  }
]
```

### Обязательные поля
| Поле | Что писать | Пример |
|------|-----------|--------|
| `name` | Название товара | `"Шоколад Kinder Chocolate"` |
| `new_price` | Цена по акции (BYN) | `"9.49"` |
| `old_price` | Обычная цена (BYN) | `"12.29"` |
| `end_date` | Дата окончания (ГГГГ-ММ-ДД) | `"2026-04-15"` |
| `category` | Категория товара | `"сладости"` |

### Как обновить
1. Откройте папку `data/promotions/`
2. Создайте новый файл или отредактируйте существующий
3. Перезапустите бота: `docker-compose restart bot`

> **Важно:** Просроченные акции (end_date в прошлом) автоматически скрываются — удалять их не нужно.

## 2. Обновление FAQ

### Где хранятся
```
data/faq/
├── general.json       — общие вопросы (доставка, возврат, оплата)
├── contacts_full.json — контактные данные, горячая линия
└── faq_eplus.docx     — FAQ по карте Еплюс (из файла от маркетинга)
```

### Формат файла
Создайте или отредактируйте `data/faq/новый_faq.json`:

```json
[
  {
    "question": "Как работает бесконтактная оплата?",
    "answer": "Во всех магазинах сети принимается бесконтактная оплата картами Visa и Mastercard, а также через Apple Pay, Google Pay и Samsung Pay. Просто приложите карту или телефон к терминалу."
  },
  {
    "question": "Есть ли доставка продуктов?",
    "answer": "Да! Доставка доступна через сервис e-доставка (edostavka.by). Оформите заказ на сайте или в приложении, выберите удобное время. Минимальная сумма заказа — 30 BYN."
  }
]
```

### Обязательные поля
| Поле | Что писать | Пример |
|------|-----------|--------|
| `question` | Вопрос покупателя | `"Как вернуть товар?"` |
| `answer` | Полный ответ | `"Возврат товара возможен в течение 14 дней..."` |

### Как обновить
1. Откройте `data/faq/`
2. Добавьте вопросы в существующий файл или создайте новый `.json`
3. Перезапустите бота и загрузите данные:
```bash
python3.11 scripts/import_alexey_data.py
docker-compose restart bot
```

## 3. Обновление рецептов

### Где хранятся
```
data/recipes/
├── sample.json            — тестовые рецепты
├── youtube.json           — рецепты с YouTube-канала Евроопт
└── recipes_alexey.docx    — рецепты из файла маркетинга (735 рецептов)
```

### Формат файла
Создайте `data/recipes/новые_рецепты.json`:

```json
[
  {
    "name": "Салат Цезарь с курицей",
    "ingredients": [
      "куриное филе 300г",
      "листья салата романо 200г",
      "сыр пармезан 50г",
      "сухарики 100г",
      "соус цезарь 3 ст.л."
    ],
    "steps": [
      "Обжарьте куриное филе на гриле до готовности, нарежьте полосками",
      "Порвите листья салата руками в большую миску",
      "Добавьте курицу, сухарики и тёртый пармезан",
      "Заправьте соусом цезарь, перемешайте"
    ],
    "time_minutes": 20,
    "servings": 2,
    "category": "салат"
  }
]
```

### Обязательные поля
| Поле | Что писать | Пример |
|------|-----------|--------|
| `name` | Название блюда | `"Борщ классический"` |
| `ingredients` | Список ингредиентов | `["свёкла 300г", "капуста 200г"]` |
| `steps` | Шаги приготовления | `["Сварите бульон", "Нарежьте овощи"]` |

### Как обновить
1. Откройте `data/recipes/`
2. Добавьте рецепты в новый `.json` файл
3. Загрузите в базу знаний и перезапустите бота:
```bash
python3.11 scripts/import_alexey_data.py
docker-compose restart bot
```

## 5. Переключение LLM-модели

Бот поддерживает переключение между языковыми моделями без остановки.

### Как переключить
Отредактируйте файл `.env`:

```bash
# Бюджетная (рекомендуемая для старта)
LLM_MODEL=openai/gpt-4o-mini

# Средняя
LLM_MODEL=deepseek-ai/deepseek-v3.2

# Премиум
LLM_MODEL=anthropic/claude-sonnet-4-20250514
```

Перезапустите: `docker-compose restart bot`

### Доступные модели

| Модель | Качество | Скорость | Стоимость |
|--------|----------|----------|-----------|
| GPT-4o-mini | 5/5 | 2 сек | Бюджетная |
| GLM-5-Turbo | 5/5 | 10 сек | Бюджетная |
| DeepSeek V3.2 | 5/5 | 18 сек | Бюджетная |
| Claude Sonnet 4 | 5/5 | 9 сек | Премиум |
| Qwen3 235B | 5/5 | 8 сек | Средняя |

## 6. Мониторинг

### Логи
Все запросы пользователей логируются в `logs/interactions.jsonl`.

Каждая запись содержит:
- Текст запроса и ответа
- Время ответа (мс)
- Использованная модель
- Количество токенов и стоимость
- Срабатывание контент-фильтра

### Ежедневный отчёт
```bash
python3.11 -m src.monitoring.daily_report
```

Генерирует отчёт в `reports/` с метриками за день.
