# LLM Router — Research дешёвых моделей для AI-помощника

**Цель:** найти модели с минимальной стоимостью и приемлемым качеством для нашей задачи (RAG + русский язык + следование длинному системному промпту, 380+ строк).

**Текущая модель в проде:** `openai/gpt-4o-mini` через OpenRouter
**Текущая стоимость:** ~$0.15/$0.60 за 1M токенов (input/output)

---

## Что важно для нашей задачи

| Критерий | Вес | Почему |
|---|---|---|
| Качество русского | 🔥🔥🔥 | 99% запросов на русском, белорусская специфика (Беларусь, города) |
| Следование SYSTEM_PROMPT | 🔥🔥🔥 | У нас 380+ строк промпта с правилами антигаллюцинаций, не должна забывать инструкции |
| Контекст ≥32K токенов | 🔥🔥 | RAG-чанки + история чата = 5-10K токенов в среднем, иногда выше |
| Latency p95 ≤3s | 🔥🔥 | Telegram UX, не любит ожиданий |
| Стоимость | 🔥🔥 | Ожидаемая нагрузка 100K юзеров → ~3M запросов/мес |
| Доступность из РБ | 🔥 | Прямой API или через relay (Render Frankfurt) |

## Риски и compromisse

### GLM-4.5-Flash (Zhipu AI) — самая дешёвая
- ✅ Очень дешёвая
- ✅ Контекст 128K
- ✅ Хороший русский (тестировали в проде ранее на стадии MVP)
- ⚠️ Серверы в КНР — тестеру может это не понравиться (политика заказчика)
- ⚠️ Менее стабильный uptime по сравнению с OpenAI
- ⚠️ Реже обновляется (последний релиз — Q1 2026)

### DeepSeek V3 — золотая середина
- ✅ Очень хороший русский
- ✅ Отлично следует длинному промпту
- ✅ Контекст 128K
- ✅ Хорошая репутация в опен-сорсе
- ⚠️ Серверы тоже в КНР
- ⚠️ В 2× дороже GLM, в 1.8× дороже gpt-4o-mini

### Mistral Small — европейский вариант
- ✅ EU-серверы
- ✅ GDPR-совместимый
- ✅ Хороший русский (Mistral специализируется на multilingual)
- ⚠️ Контекст только 32K
- ⚠️ Мене популярен, меньше community support

### YandexGPT 5 Lite — для контура РБ/РФ
- ✅ Серверы в РФ, доступен из РБ напрямую без relay
- ✅ Отличный русский, понимает белорусские топонимы
- ⚠️ Документация на русском, мало community
- ⚠️ Не всегда отвечает по структуре, иногда «отсебятина»

## Стратегия router'а

### Уровень 1: Smart cost-optimization

```python
# settings.llm_router_strategy = "cost_optimized"
# Простые FAQ запросы (короткий ответ < 200 токенов) → GLM-4.5-Flash
# Сложные RAG запросы (>5 источников) → DeepSeek V3
# С большим контекстом (>20K) → gpt-4o-mini (надёжно)
# PII / content_filter / canonical — уже есть, до LLM не доходят
```

### Уровень 2: Quality fallback

Если основная модель вернула:
- Слишком короткий ответ (<50 символов)
- Содержит маркеры галлюцинации (пустой список после preamble, грамматический мусор)
- Timeout >5s

→ автоматически перевыпустить запрос на gpt-4o-mini.

### Уровень 3: A/B тестирование

20% трафика на «дешёвую» модель, 80% на baseline. По метрикам:
- Cost per query
- Avg latency
- User satisfaction (длина диалога, повторные вопросы)

Решение: оставлять или откатывать.

## Бенчмарк-методика

Для сравнения моделей используем **наши 59 acceptance-сценариев** (`tests/scenarios/tester_scenarios.json`):

```bash
python3.11 scripts/benchmark_llm_models.py \
    --models gpt-4o-mini deepseek-chat glm-4-flash qwen2.5-72b \
    --runs 3  # 3 прогона для стабильности
```

**Метрики:**
1. **Pass rate** — сколько ответов проходят валидатор
2. **Latency** — p50, p95
3. **Cost** — расчёт по реальным token usage
4. **Consistency** — насколько одинаково отвечает на повторение того же вопроса (3 прогона)
5. **Human review** — топ-10 спорных кейсов читаем глазами

- Запустить ночью (за 1-2 часа) в shadow-режиме
- Сделать таблицу сравнения

### Фаза 3: A/B (3-5 рабочих дней)
- В `pipeline.py` добавить рандомное переключение модели для 20% трафика
- Логировать `model_used`, `latency_ms`, `cost_estimate` в `RequestLog`
- 1 неделя сбора данных

### Фаза 4: Decision (1 день)
- Если победитель ясен → переключить prod на него
- Если нет — оставить текущую конфигурацию + сообщить заказчику

### Фаза 5: Smart routing (опц., если есть смысл) (3-5 дней)
- Реализовать классификатор «сложный vs простой запрос»
- Простые → дешёвая модель, сложные → топовая

|---|---|

## Outcomes which we want

1. **Снижение стоимости на 30-50%** при сохранении качества (≥90% pass rate на тестах)
2. **Снижение latency** (если новая модель быстрее)
3. **Резервный канал** — если основной провайдер недоступен, автоматический fallback
