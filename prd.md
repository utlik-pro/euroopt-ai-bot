# PRD: AI-помощник для покупателей Евроопт — MVP

## Обзор проекта

Telegram-бот для розничных сетей «Евроопт», «Грошык» и «Хит Дискаунтер» (ООО «Евроторг»), который подсвечивает актуальные акции и офферы, отвечает на FAQ и рекомендует рецепты со ссылками на покупку — без сбора и обработки персональных данных.

**Заказчик:** ООО «Евроторг»
**Исполнитель:** ООО «БелХард Девелопмент»
**Договор:** № 1903/26-01 от 19.03.2026
**Сумма:** 50 000 BYN (25К аванс получен 31.03.2026)
**Дедлайн:** 12 рабочих дней от 31.03.2026 = ~16 апреля 2026
**Вариант:** А — Экспресс (2 недели, 2 разработчика параллельно)

## Ключевая ценность

Бот — это канал коммуникации акций, а не справочник. Каждый ответ бота подсвечивает актуальные предложения сети в контексте запроса покупателя:

- «Что приготовить на ужин?» → рецепт + «Куриное филе сейчас по акции — 8.90 вместо 12.90 BYN»
- «Что есть интересного?» → подборка лучших акций недели
- «Хочу борщ» → рецепт + «Свёкла по 2.25 — акция до пятницы»

**Первый AI-бот в ритейле Беларуси.**

## Архитектура

```
Покупатель → Telegram → Backend (сервер в РБ) → RAG → LLM API
                                    ↓
                               Логи (РБ)
```

### 3 слоя обработки запросов

1. **Контент-фильтр** — отсечение запросов на политические и религиозные темы (до обработки)
2. **RAG (база знаний)** — запросы по Евроторгу идут в базу знаний (FAQ, акции, рецепты), затем ответ через LLM
3. **Общий доступ к LLM** — запросы НЕ связанные с Евроторгом обрабатываются напрямую через LLM (как ChatGPT с доп. знаниями)

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| LLM | Claude API (старт), переключаемый адаптер на DeepSeek/GLM |
| RAG | Embeddings + векторная БД (ChromaDB или Qdrant) |
| Контент-фильтр | Правила + промпт-инжиниринг |
| Backend | Python (FastAPI) |
| Telegram | aiogram 3.x |
| Хостинг | VPS в контуре РБ |
| CI/CD | GitHub Actions или аналог |
| Логирование | Структурированные логи, метрики использования |

## Функционал MVP — 9 компонентов

### 1. Telegram-бот
- aiogram 3.x
- Диалоговые сценарии, команды (/start, /help, /акции, /рецепты)
- Приветственное сообщение с описанием возможностей
- Обработка текстовых сообщений
- Graceful error handling (при недоступности LLM — вежливое сообщение)

### 2. RAG-система
- Embeddings для индексации базы знаний
- Векторная БД для хранения и поиска
- Индексация FAQ, рецептов, акций, информации о магазинах
- Ранжирование результатов по релевантности
- Автоматическое обновление индекса при изменении данных

### 3. LLM-адаптер (мультимодельная архитектура)
- Абстракция для переключения между моделями
- Claude API на старте (лучшее качество на русском)
- DeepSeek API как альтернатива (переключение за 1 день)
- Промпт-инжиниринг: системный промпт с характером бота, правилами фильтрации
- Настраиваемые параметры (temperature, max_tokens)
- A/B тестирование между моделями — опционально

### 4. База знаний: FAQ
- Структурирование данных от заказчика
- Категории: режим работы, доставка, возврат, оплата, программа Е+
- Загрузка в RAG-систему
- Формат хранения: markdown/JSON
- CLI или скрипт для обновления

### 5. База знаний: Рецепты
- 10–30 рецептов от заказчика
- Структура: название, ингредиенты, шаги, время
- Ссылки на e-доставку для ингредиентов
- Теги: завтрак/обед/ужин, быстрый/сложный, бюджетный

### 6. Промоушн-движок (Акции)
- Загрузка актуальных акций (ручное обновление через файл/скрипт)
- Подсветка офферов в контексте любого запроса
- Логика: если в ответе упоминается продукт по акции — добавить информацию
- Срок действия акций (auto-expire)
- Формат: название, старая цена, новая цена, срок, магазины

### 7. Контент-фильтр
- Блокировка: политика, религия, конкуренты, нецензурная лексика
- Правила на уровне промпта + regex pre-check
- Вежливый отказ с перенаправлением на тематику сети
- Логирование заблокированных запросов

### 8. Деплой и мониторинг
- Docker-контейнеры
- VPS в контуре РБ
- CI/CD pipeline
- Логирование всех запросов и ответов
- Метрики: количество запросов, время ответа, популярные темы
- Алерты при сбоях
- Бэкапы данных

### 9. Документация
- Инструкция для маркетинга: как обновлять FAQ, рецепты, акции
- Техническая документация: архитектура, деплой, конфигурация
- README проекта

## Сценарий работы (flow)

1. Клиент пишет запрос в Telegram-бот
2. Контент-фильтр проверяет запрос (политика/религия → вежливый отказ)
3. Если запрос связан с Евроторгом → RAG ищет в базе знаний → LLM формулирует ответ с подсветкой акций
4. Если общий запрос → LLM отвечает напрямую
5. Бот отправляет ответ пользователю
6. Запрос и ответ логируются
